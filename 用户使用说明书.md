# CHM2MD 用户使用说明书

## 软件简介

CHM2MD 是一个将 CHM (Compiled HTML Help) 格式文档转换为 Markdown 格式的命令行工具。该工具专为处理中文 CHM 文档优化，支持层级合并、自动分片等高级功能。

## 功能特性

- CHM 文件反编译
- 自动识别并解析 HHC 目录结构
- HTML 到 Markdown 的高质量转换
- 支持两种输出模式：
  - **默认模式**：保留原始结构，每个 HTML 文件转换为独立 Markdown 文件
  - **合并模式**：按指定层级合并多个文档，生成更易阅读的合并文件
- 自动分片超大文件（单文件超过 48 万字符自动分割）
- 支持中文 GBK 编码自动识别与转换
- 智能过滤导航链接、版权声明等无关内容

## 系统要求

- **操作系统**：Windows（依赖系统自带的 `hh.exe` 工具）
- **运行环境**：Go 1.24.0 或更高版本（如使用源码编译）

## 安装方法

### 方式一：使用预编译版本

直接使用 `build/chm2mdGo.exe` 可执行文件。

### 方式二：从源码编译

```bash
# 克隆项目
git clone <仓库地址>
cd chm2md

# 编译
go build -o chm2md.exe .

# 或直接运行
go run main.go [参数]
```

## 命令行参数

| 参数 | 简写 | 默认值 | 说明 |
|------|------|--------|------|
| `-in` | | (必填) | 输入的 CHM 文件路径 |
| `-out` | | `./output` | 输出目录路径 |
| `-level` | | `0` | 合并层级：0=不合并(默认)，1=按根目录合并，2=按二级目录合并... |

## 使用示例

### 示例 1：基本使用（默认模式）

```bash
chm2md.exe -in "文档.chm"
```

将生成：
```
output/
├── temp_source/     # 反编译的临时源文件
└── markdown/        # 转换后的 Markdown 文件
    ├── SUMMARY.md   # 目录索引文件
    ├── 章节1.md
    ├── 章节2.md
    └── ...
```

### 示例 2：指定输出目录

```bash
chm2md.exe -in "文档.chm" -out "./mydocs"
```

### 示例 3：合并模式（按根目录合并）

```bash
chm2md.exe -in "文档.chm" -level 1
```

将同一根目录下的所有文档合并为一个 Markdown 文件。

### 示例 4：合并模式（按二级目录合并）

```bash
chm2md.exe -in "文档.chm" -level 2
```

### 示例 5：完整命令

```bash
chm2md.exe -in "D:\文档\帮助文档.chm" -out "D:\输出" -level 1
```

## 输出模式详解

### 默认模式 (level=0)

- 每个 HTML 页面转换为独立的 `.md` 文件
- 保持原始 CHM 的目录层级结构
- 生成 `SUMMARY.md` 索引文件，可用于 GitBook 等工具

**适用场景**：
- 需要保持原始文档结构
- 文档内容较多，需要分章节查阅
- 用于构建在线文档系统

### 合并模式 (level≥1)

- 按指定层级将多个文档合并为单一文件
- 自动生成层级标题
- 内容过大时自动分片

**适用场景**：
- 需要生成完整手册
- 离线阅读或打印
- 内容整合归档

**分片规则**：
- 单文件超过 480,000 字符时自动分割
- 分片文件命名格式：`文件名1.md`、`文件名2.md`...

## 技术原理

### 处理流程

```
CHM 文件
    │
    ▼
[反编译] 使用 hh.exe -decompile 提取内容
    │
    ▼
[解析 HHC] 解析目录结构，构建文档树
    │
    ▼
[转换模式选择]
    ├─→ 默认模式：逐文件转换 + 生成 SUMMARY.md
    └─→ 合并模式：按层级合并 + 自动分片
    │
    ▼
Markdown 文件
```

### 核心模块

#### 1. 反编译模块 (pkg/decompiler)

- 调用 Windows 系统的 `hh.exe` 工具
- 处理特殊字符（中文、空格、&符号等）
- 采用临时文件策略确保兼容性

#### 2. 解析模块 (pkg/parser)

- 解析 HHC (HTML Help Contents) 目录文件
- 构建文档树结构（节点包含标题、路径、子节点）
- 支持多种 HTML 结构变体

#### 3. 编码模块 (pkg/encoding)

- 自动检测并转换 GBK 编码
- 解决中文 CHM 的乱码问题

#### 4. 转换模块 (pkg/converter)

- HTML 到 Markdown 转换
- DOM 清洗：移除脚本、样式、图片、隐藏元素
- 表格预处理：处理表格内段落，防止格式破裂
- 链接简化：将超链接转换为纯文本格式
- 后处理过滤：移除"父主题"、版权声明、导航链接等

#### 5. 合并模块 (pkg/merger)

- 递归遍历文档树
- 按层级合并内容
- Unicode 字符计数，精准分片

## 常见问题

### Q1：提示"缺少 hh.exe"

**原因**：本工具依赖 Windows 系统自带的 `hh.exe` 程序。

**解决方案**：
- 确保在 Windows 系统上运行
- 检查系统环境变量是否正确配置

### Q2：输出内容为空或乱码

**原因**：CHM 文件编码格式不标准。

**解决方案**：
- 工具默认尝试 GBK 解码
- 如果仍然乱码，可能是非中文 CHM，需要修改源码适配

### Q3：合并后文件过大

**解决方案**：工具已内置自动分片功能，单个文件超过 48 万字符会自动分割为多个文件。

### Q4：如何清理临时文件？

**解决方案**：转换完成后可手动删除 `输出目录/temp_source` 文件夹。

```

├── go.mod                     # Go 模块定义
├── go.sum                     # 依赖版本锁定

│   ├── decompiler/
│   │   └── hh.go              # CHM 反编译
│   ├── parser/
│   │   ├── hhc.go             # HHC 文件解析
│   │   └── structure.go       # 数据结构定义
│   ├── converter/
│   │   └── markdown.go        # HTML 转 Markdown
│   ├── merger/
│   │   └── builder.go         # 合并与分片
│   └── encoding/
│       └── decoder.go         # 编码转换
└── build/
    └── chm2mdGo.exe           # 编译后的可执行文件
```

## 注意事项

1. **仅支持 Windows**：依赖 `hh.exe` 工具，无法在 Linux/macOS 上运行
2. **中文优化**：针对中文 CHM 文档优化，其他语言可能需要调整编码处理
3. **图片处理**：默认移除图片标签，不提取图片资源
4. **超大文件**：合并大文档时建议使用 `-level` 参数控制输出粒度
